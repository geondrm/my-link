# 마이링크 (MyLink) - 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요
- **프로젝트명**: 마이링크 (MyLink)
- **목적**: 여러 개의 링크(SNS, 블로그, 포트폴리오 등)를 하나의 프로필 페이지와 단일 URL로 통합하여 손쉽게 공유할 수 있는 웹 서비스 제공.
- **대상 사용자**: 
  - 여러 채널을 운영하는 인플루언서 및 크리에이터
  - 포트폴리오를 한 곳에 모아두고 싶은 취업준비생 및 프리랜서
  - 이벤트 페이지 등을 하나의 링크로 모아서 홍보하려는 소상공인 및 기업
- **서비스 정책**: 런칭 시점에는 모든 기능을 제한 없이 사용할 수 있는 **완전 무료 단일 플랜**으로 서비스 오픈.

## 2. 핵심 기능 목록 (필수 기능)

1. **사용자 인증 및 계정 관리**
   - 이메일 기반 회원가입, 로그인, 로그아웃 기능.

2. **고유 프로필 URL 제공**
   - 사용자마다 식별 가능한 단일 링크 생성.
   - 별도의 ID(핸들)나 `@` 기호를 요구하지 않고 사용자의 **닉네임(`displayname`)을 URL 슬러그(Slug)로 사용하여** 그대로 깔끔한 접속 URL을 구성함 (예: `domain.com/displayname`).
   - **기본값 자동 설정**: 회원가입 시 사용하는 이메일 주소(예: 구글 지메일)의 앞부분(ID)을 추출하여 `displayname`의 초기값으로 자동 설정함.
   - 닉네임이 주소가 되므로 회원가입 및 수정 시 **닉네임 중복 검사**를 필수로 수행.
   - **예약어 방지 로직**: 라우팅 충돌을 막기 위해 `login`, `admin`, `api`, `dashboard` 등의 시스템 페이지 이름은 닉네임으로 생성할 수 없도록 제한함.

3. **링크 관리 기능 (대시보드)**
   - 새로운 외부 링크(이름, URL)를 추가, 수정, 삭제할 수 있음.
   - 모든 링크 정보 수정은 페이지 이동이나 팝업창 없이 클릭하여 텍스트를 바로 변경하는 **인라인 편집(Inline Editing)** 방식으로 구현함.
   - **자동 파비콘 연동**: **구글 파비콘 API(Google Favicon API)**를 사용하여 사용자가 등록한 URL 도메인의 파비콘을 추출해 링크 항목의 아이콘으로 자동 표시함.
   - *(주의: 링크 활성화/비활성화 스위치 기능 및 드래그 앤 단롭을 통한 순서 변경 기능은 제공하지 않음)*

4. **미디어 임베드 기능**
   - 단순 외부 링크 이동뿐만 아니라, 유튜브 영상이나 이미지 등을 프로필 페이지에 직접 임베드하여 재생 및 표시.

5. **프로필 설정**
   - URL 슬러그로 사용되는 닉네임(`displayname`) 외에 프로필 화면에 표시될 **실제 사용자의 이름(`username`)**을 별도로 설정함.
   - 유저네임(`username`)과 자신을 소개하는 짧은 텍스트(Bio)를 최초 작성 후 자유롭게 **수정 가능**해야 함. (프로필 정보 수정 역시 가급적 인라인 편집을 사용)
   - *(주의: 프로필 이미지 업로드 기능은 제공하지 않음)*

6. **프로필 페이지 렌더링**
   - 누구나 접속할 수 있는 퍼블릭 페이지에서 등록된 프로필(실제 이름인 `username`, Bio)과 링크, 임베드 콘텐츠를 랜더링함.
   - 모바일 환경에 최적화된 세로형 UI로 제공.
   - *(주의: 테마 변경 기능, 대시보드 내 모바일 뷰 미리보기 화면은 제공하지 않음)*

## 3. 향후 추가 예정 기능
- **링크 클릭 조회수**: 추후 업데이트를 통해 각 개별 링크를 방문자가 얼마나 클릭했는지 추적하고 보여주는 기능을 추가할 예정. *(단, 전체 방문자 통계 기능은 구현하지 않음)*

## 4. 데이터베이스 모델링 주요 사항
- 사용자 데이터 및 상태 저장을 위해 NoSQL 기반의 데이터베이스 구조를 고려함.
- **필수 필드 포함**: 사용자 컬렉션 문서에는 URL 매핑을 위한 `displayname`과 실제 화면에 보여줄 `username` 필드가 모두 필수적으로 포함되어야 함.
- **서브 컬렉션(Sub-collection) 활용**: 사용자의 링크 목록이나 관련 데이터를 저장할 때 메인 유저 문서 하위의 서브 컬렉션 방식을 사용하여 데이터를 관리함.
